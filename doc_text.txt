Sumário



1. Visão Geral
O Montador Pro é uma plataforma digital completa criada para a NF Móveis Planejados — empresa de marcenaria e instalação de móveis planejados com atuação em Goiás. O sistema digitaliza e automatiza todo o processo que vai desde a criação de uma ordem de serviço no escritório até a conclusão e avaliação da instalação pelo montador em campo.
A plataforma é composta por quatro módulos integrados que se comunicam por meio de uma API central:

🌐

Landing Page

Site institucional da NF Móveis. Apresenta serviços ao cliente final e serve como porta de entrada para o app dos montadores.
📱

App do Montador

PWA instalável no celular. Recebe convites, registra chegada com GPS, envia fotos e finaliza serviços em campo.
🔧

Painel Admin

Dashboard web para o escritório. Cria OS, localiza montadores por raio, aprova cadastros e acompanha execuções.
⚙️

API Backend

Servidor Node.js com banco PostgreSQL. Centraliza autenticação, lógica de negócio, push notifications e armazenamento.

🎯
Objetivo: eliminar a alocação manual de montadores, reduzir o tempo entre a abertura de uma OS e o início da execução, e garantir rastreabilidade completa de cada instalação — do convite ao montador até a avaliação final.

2. Fluxo de Operação
Cada serviço percorre um ciclo completo de 8 etapas, desde a criação da OS no escritório até a avaliação final:

#
Etapa
O que acontece
1
🗂️  Criação da OS
O admin preenche o formulário no painel: nome e contato do cliente, endereço da obra, tipo de projeto, valor do montador e data agendada.
2
📍  Geolocalização
O sistema geocodifica o endereço automaticamente e calcula a distância de todos os montadores ativos pelo algoritmo de Haversine.
3
📨  Envio de Convites
Os 3 montadores mais próximos dentro do raio configurado recebem convite simultâneo via push notification e link de WhatsApp. Prazo: 20 minutos.
4
✅  Aceite
O montador aceita pelo app. A transação é atômica no banco — apenas um montador pode aceitar cada OS, evitando conflito.
5
📍  Chegada
No local, o montador toca "Confirmar Chegada". O backend valida a posição GPS (raio de 200 m) e o horário (±15 min do agendado).
6
📸  Fotos
O montador fotografa mínimo 4 etapas obrigatórias: Antes, Durante, Detalhe e Finalizado. As fotos são enviadas para o servidor em tempo real.
7
🏁  Finalização
O montador finaliza o serviço pelo app. O sistema valida as fotos, registra a hora de conclusão e libera o processo de pagamento.
8
⭐  Avaliação
O admin avalia o serviço (1–5 estrelas). A nota média do montador é recalculada automaticamente, influenciando a prioridade em futuros convites.

4. Funcionalidades Técnicas
4.1  Geolocalização e Despacho Automático
Ao criar uma OS, o sistema executa três operações em sequência:
Geocodificação do endereço da obra via OpenStreetMap Nominatim (gratuito, sem API key)
Cálculo de distância por Fórmula de Haversine entre a obra e o GPS registrado de cada montador
Ordenação por proximidade e seleção dos 3 mais próximos dentro do raio (10 / 20 / 30 / 50 / 100 km)

📍
Os convites são enviados simultaneamente para os 3 montadores. O primeiro a aceitar "trava" a OS via transação atômica no PostgreSQL — garantindo que não haja duplo aceite.

4.2  Push Notifications (VAPID)
O sistema usa o protocolo Web Push com criptografia VAPID para enviar notificações ao dispositivo do montador mesmo com o app fechado, sem custo e sem Firebase. Os eventos notificados são:
Novo convite de serviço — instantâneo, com valor e endereço da obra
Broadcast do admin — mensagem personalizada para todos os montadores ativos

4.3  Autenticação e Segurança

Mecanismo
Como funciona
JWT Access Token
Tokens de curta duração (15 min). Cada requisição autenticada deve enviar no header Authorization.
JWT Refresh Token
Token de 30 dias. Renova automaticamente o access token sem exigir novo login.
bcrypt (rounds: 12)
Hashing das senhas com custo alto — resiste a ataques de força bruta e rainbow tables.
Rate Limiting
Limite de tentativas por IP — previne ataques de enumeração de senhas e DDoS.
Erro genérico no login
Retorna sempre a mesma mensagem para senha errada ou usuário inexistente (timing-safe).
x-admin-secret
Header especial separado da auth dos montadores — isola completamente o acesso administrativo.

4.4  Validação de Chegada com GPS
Quando o montador toca "Confirmar Chegada", o backend realiza duas validações simultâneas:
Janela de tempo: o montador deve estar dentro de ±15 minutos do horário agendado
Raio geográfico: a coordenada GPS do celular deve estar a menos de 200 metros da obra

⚠️
Ambas as validações ocorrem exclusivamente no backend — não no app — prevenindo adulteração das coordenadas ou do horário pelo dispositivo do usuário.

8. Infraestrutura e Deploy
8.1  Arquitetura de Produção
O sistema roda em três camadas, todas orquestradas pelo Docker Compose e provisionadas no Railway:

                   Internet (usuário / montador)
                          |
             ┌────────────▼──────────────┐
             │       NGINX  (porta 80)    │
             │  /        → index.html     │
             │  /app/    → app.html       │
             │  /admin/  → admin-v2.html  │
             │  /api/    → proxy :3000    │
             └────────────┬──────────────┘
                          |
             ┌────────────▼──────────────┐
             │  Node.js Backend  :3000   │
             │  Express + Prisma + JWT   │
             └────────────┬──────────────┘
                          |
             ┌────────────▼──────────────┐
             │   PostgreSQL  :5432       │
             └───────────────────────────┘

8.2  Railway
Plataforma cloud usada para deploy do backend e banco de dados. A cada push no GitHub, um novo deploy é iniciado automaticamente. Recursos incluídos:
PostgreSQL gerenciado com backups automáticos e variável DATABASE_URL injetada diretamente
HTTPS automático no domínio *.up.railway.app com certificado SSL gratuito
Logs em tempo real, métricas de uso e terminal de acesso ao container
Variáveis de ambiente seguras — nunca expostas no código-fonte ou repositório

8.3  Resend (E-mail)
Serviço de e-mail transacional usado para notificações automáticas: convites de serviço, confirmação de aprovação do montador e avisos de OS concluída. Plano gratuito inclui 3.000 e-mails/mês.
